## 1. 프로젝트 개요

### 1.1 프로젝트 주제 / 기획 의도

<b>프로젝트 주제</b><br>
- 웹 접근성과 사용자 편의성을 고려한 커피숍 웹사이트 개발 

<b>기획 의도</b><br>
- 사용자 친화적인 웹페이지를 통해 제품 소개, 검색 기능, 멤버 정보를 제공하고 사용자와의 유대감 형성

<b>특화 포인트</b><br>
- Figma 디자인 기반 UX를 고려하여 반응형 및 서브페이지 기획

<b>유사 서비스와 차별점</b><br>
- 모바일 반응형 최적화로 다양한 기기에서의 접근성 향상

### 1.2 프로젝트 내용
내용<br>
<ul>
<li>HTML5, CSS를 사용하여 Figma 디자인 구현</li>
<li>메인 페이지, 원두 검색 페이지, 장바구니 페이지, 직원 소개 페이지 구현</li>
<li>깃허브(GitHub)를 통한 형상 관리</li>
</ul>
컨셉<br>
<ul>
<li>사용자 친화적 경험 제공</li>
<li>직관적인 UX/UI 디자인</li>
</ul>

원두 검색 페이지<br>
- 직관적인 UX 제공

멤버 소개 페이지<br>

장바구니 페이지<br>
- 원두제품 선택 및 결제 페이지 제공

## 5. 자체 평가 의견

### 5.1 👍 잘한 점
<ol>
 <li>Github organization을 통한 공동 작업 및 관리</li>
 <li>팀원 개인 모두 Merge 경험 및 Conflict 해결</li>
 <li>팀원들이 더 나은 구현을 위해 지속적인 소통 및 issue 공유</li>
 <li>기존 탬플릿 디자인을 최대한 해치지 않는 선에서 작업</li> 
</ol>

### 5.2 👎 아쉬운 점
<ol>
 <li>메인페이지 구현에 시간을 많이 투자해 추가적인 서브페이지 기획 및 구현의 완성도를 높이지 못했던 점</li>
 <li>코드 주석 작업이 미흡했던 점</li>
 <li>Github 컨벤션 메세지의 통일성 문제(2차 프로젝트에 보완할 예정)</li>
</ol>

### 5.3 👏 개선할 점
<ol>
 <li>추가적인 서브페이지 기획 및 구현을 위한 작업</li>
 <li>태블릿 기준 media query에 대한 보완</li>
 <li>컨텐츠 display에 대한 정확한 위치 조정</li>
 <li>화면 비율이 작아졌을 때 비율에 대한 반응형 작업</li>
</ol>
